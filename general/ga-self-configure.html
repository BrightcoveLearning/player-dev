Introduction

The configuration process consists of the following tasks:

  • Select the events to track from those supported by Brightcove Player.
  • Select the custom dimensions from those supported by Google Analytics.
  • In Google Analytics, create the selected custom dimension.
  • In Video Cloud Studio, add configuration information about the Google Analytics plugin using the player's JSON editor.

Select events

In the JSON configuration you will list the events you wish to track. Decide which of the following you to track, and they will be used later:

  • player_load
  • video_engagement
  • video_impression
  • play_request
  • ad_start
  • ad_end
  • error
  • video_view
  • video_complete

Select custom dimensions

There are eight custom dimensions from which you can choose to use in your Google Analytics data. Decide which of the following to use, and they will be used later:

  • bcvideo_account
  • bcvideo_platform_version
  • bcvideo_player
  • bcvideo_range
  • bcvideo_session
  • bcvideo_tags
  • bcvideo_video_duration
  • bcvideo_video_seconds_viewed

Create custom dimensions

For each of the custom dimensions you are using, you must create. The steps in this document from Google lead you through the process: Create and edit custom dimensions and metrics.

Add JSON configuration to player

You've made the decisions about what events to use, and what custom dimensions to use, and created the latter. You are know ready to build the JSON that you will include in the player configuration.

{
  "registry_id": "@brightcove/videojs-bc-ga",
  "version": "1.x",
  "options": {
    "paramsToTrack": {
    "bcvideo_video_seconds_viewed": "dimension1",
    "bcvideo_range": "dimension2",
    "bcvideo_video_duration": "dimension3",
    "bcvideo_player": "dimension4",
    "bcvideo_session": "dimension5"
  },
  "eventsToTrack": {
    "video_impression": "Video Impression",
    "play_request": "Play Request",
    "video_engagement": "Video Engagement"
  },
  "tracker": "UA-123456-1"
  }
}
